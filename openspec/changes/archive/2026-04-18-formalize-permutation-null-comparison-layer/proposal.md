# Proposal: formalize-permutation-null-comparison-layer

## Boundary problem

- AlphaForge already has a stabilized MA search, evidence, and post-search policy stack, but it still lacks a first explicit anti-overfitting diagnostic.
- There is no canonical layer that compares a real MA candidate against a null distribution generated by permutation while keeping candidate parameters fixed.
- If this diagnostic is added ad hoc inside CLI, runner, or reporting code, it will quickly become a second owner of scoring or verdict semantics.

## Canonical ownership decision

- `src/alphaforge/permutation.py` will own the null-comparison computation for the current MA workflow.
- `src/alphaforge/schemas.py` will own the runtime summary and receipt contracts for the permutation test output.
- `src/alphaforge/storage.py` will own persisted filenames, summary JSON, and permutation-score artifact layout.
- `src/alphaforge/cli.py` will expose a thin `permutation-test` command that assembles request DTOs and prints the upstream summary payload.

## Scope

- Compare one fixed MA candidate against a permutation-generated null distribution.
- Permute market-data rows as whole units so OHLCV row coherence is preserved.
- Keep candidate parameters fixed across permutations.
- Use one target metric only for the first pass: `score`.
- Compute an empirical p-value from the observed score and the permutation scores.
- Persist a compact summary JSON and a stable score-list artifact when an output directory is provided.

## Out of scope

- Genetic algorithms
- Random search
- Additional strategy families
- White’s Reality Check, SPA, bootstrap families, or multiple-testing correction frameworks
- Policy verdict changes
- Broad report or visualization redesign

## Acceptance conditions

- The repository exposes a deterministic, seed-controlled permutation/null-comparison diagnostic.
- The null-comparison contract is explicit about what is permuted and what remains fixed.
- Output artifacts have stable names and a documented summary shape.
- Tests cover deterministic behavior, empirical comparison semantics, and CLI/storage output consistency.
