# Delta for Permutation Null Comparison Contract

## MODIFIED Requirements

### Requirement: permutation semantics preserve local temporal structure via contiguous block shuffling

The null distribution SHALL be generated by converting ordered OHLCV rows into relative OHLC movement rows, splitting those relative rows into contiguous non-overlapping blocks, shuffling block order, and reconstructing a continuous synthetic price path from the original starting price while keeping candidate parameters and backtest configuration fixed.

#### Null model

- the canonical null model SHALL be named `return_block_reconstruction`
- the first market-data row SHALL be kept as the synthetic path anchor
- for each row after the first row, relative movement SHALL be computed against the previous close:
  - `open_rel = open_t / close_{t-1}`
  - `high_rel = high_t / close_{t-1}`
  - `low_rel = low_t / close_{t-1}`
  - `close_rel = close_t / close_{t-1}`
- contiguous blocks SHALL be formed from these relative rows
- rows within each relative block SHALL preserve their original order
- block order SHALL be shuffled only
- volume values MAY be permuted with the same relative rows
- synthetic OHLC values SHALL be reconstructed from the previous synthetic close:
  - `synthetic_open = previous_synthetic_close * open_rel`
  - `synthetic_high = previous_synthetic_close * high_rel`
  - `synthetic_low = previous_synthetic_close * low_rel`
  - `synthetic_close = previous_synthetic_close * close_rel`
- reconstructed OHLC values SHALL be finite and positive
- reconstructed `high` SHALL be at least the maximum of reconstructed open, high, low, and close
- reconstructed `low` SHALL be at most the minimum of reconstructed open, high, low, and close

#### Permutation contract

- the permutation unit is a contiguous block of relative OHLC movement rows
- blocks SHALL be formed from the ordered relative rows in row order
- a trailing partial block is allowed and SHALL remain as its own block
- row count SHALL be preserved
- canonical OHLCV columns SHALL be preserved
- datetime ordering shape SHALL be preserved from the original market data
- the candidate SHALL be evaluated on each reconstructed synthetic frame without reselecting parameters
- the block size SHALL be a positive integer
- the block size SHALL not exceed the number of loaded market-data rows
- the seed SHALL be explicit so the same inputs produce the same null distribution

#### Scenario: return-block reconstruction avoids absolute price stitching

- GIVEN market data containing two flat absolute price regimes at materially different price levels
- WHEN relative movement rows are block-permuted and reconstructed from the original anchor price
- THEN the synthetic path SHALL remain continuous from the previous synthetic close
- AND it SHALL NOT introduce a jump solely because an absolute high-price block followed a low-price block

### Requirement: the diagnostic summary is compact, stable, and storage-backed

The permutation diagnostic SHALL produce a stable summary artifact with explicit comparison fields, null-model metadata, and storage-owned artifact references.

#### Summary fields

- strategy name
- strategy parameters
- target metric name
- null model
- permutation mode
- block size
- real observed metric value
- permutation metric values
- permutation count
- seed
- null ge count
- empirical p-value
- artifact paths when produced

#### Scenario: summary identifies null construction method

- GIVEN a permutation diagnostic run completes
- WHEN the runtime summary is returned or serialized
- THEN it SHALL include `null_model`
- AND `null_model` SHALL equal `return_block_reconstruction`
- AND `block_size`, `permutation_count`, and `seed` SHALL remain explicit in the summary
